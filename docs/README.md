# Documentación de Aprendiendo JavaScript

Mapa de la documentación del proyecto. Esto **no** es el contenido del curso (eso vive en
los módulos numerados `01-…`/`08-…` y en [`proyectos/`](../proyectos)); aquí se documenta
**cómo se construye y mantiene** el curso. Empieza por aquí para saber qué documento
responde cada pregunta.

| Documento                                                      | Pregunta que responde                       | Cuándo leerlo                       |
| -------------------------------------------------------------- | ------------------------------------------- | ----------------------------------- |
| [`estructura-del-contenido.md`](estructura-del-contenido.md)   | ¿Cómo se organizan módulos y lecciones?     | Antes de crear o mover contenido    |
| [`conventions/`](conventions/README.md)                        | ¿Cómo escribimos y verificamos contenido?   | Antes de escribir una lección       |
| [`decisions/`](decisions/README.md)                            | ¿Por qué tomamos cada decisión?             | Antes de re-debatir algo            |
| [`product/roadmap.md`](product/roadmap.md)                     | ¿Qué contenido falta y hacia dónde va?      | Para conocer prioridades            |
| [`glossary.md`](glossary.md)                                   | ¿Qué significa cada término de JavaScript?  | Ante vocabulario desconocido        |

## Cómo mantener esta documentación

- Actualiza la línea **"Última actualización"** al editar un documento.
- Registra las decisiones relevantes como [ADRs](decisions/README.md).
- Mantén este índice al día si agregas o quitas documentos.
