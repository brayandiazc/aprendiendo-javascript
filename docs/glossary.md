# Glosario

Términos de JavaScript usados a lo largo del curso. Mantén las definiciones breves y
consistentes con cómo se explican en las lecciones. Ordenado alfabéticamente.

| Término                | Definición breve                                                                                  |
| ---------------------- | ------------------------------------------------------------------------------------------------- |
| **Arrow function**     | Función con sintaxis `=>`; no liga su propio `this`. Ver módulo 05.                                |
| **Array**              | Estructura ordenada e indexada para guardar múltiples valores. Ver módulo 05.                      |
| **Asincronía**         | Ejecución que no bloquea: el resultado llega después (callbacks, Promises, `async`/`await`).      |
| **Booleano**           | Tipo primitivo con dos valores: `true` y `false`.                                                 |
| **Callback**           | Función que se pasa como argumento para ejecutarse más tarde.                                      |
| **Clase**              | Plantilla para crear objetos con `class`, introducida en ES6. Ver módulos 06 y 07.                |
| **Coerción**           | Conversión automática (implícita) o manual (explícita) de un tipo de dato a otro.                 |
| **`const`**            | Declara una variable de asignación única (no reasignable). Ver módulo 02.                          |
| **DOM**                | *Document Object Model*: representación en árbol de una página que JS puede manipular. Módulo 08.  |
| **Desestructuración**  | Sintaxis para extraer valores de arrays u objetos en variables. Ver módulo 06.                     |
| **Encapsulamiento**    | Ocultar el estado interno de un objeto y exponer solo lo necesario. Ver módulo 07.                 |
| **ECMAScript (ES)**    | Especificación que estandariza JavaScript; ES6 (2015) introdujo `let`/`const`, clases, etc.       |
| **Evento**             | Señal de algo ocurrido en la página (clic, tecla…) a la que se reacciona con un *listener*.        |
| **Función**            | Bloque de código reutilizable que puede recibir parámetros y devolver un valor. Ver módulo 04.    |
| **Herencia**           | Mecanismo por el que una clase reutiliza y extiende a otra (`extends`, `super`). Ver módulo 07.   |
| **Hoisting**           | Comportamiento por el que las declaraciones se "elevan" al inicio de su ámbito.                   |
| **`let`**              | Declara una variable reasignable con ámbito de bloque. Ver módulo 02.                              |
| **localStorage**       | Almacenamiento clave-valor persistente en el navegador. Ver módulo 08.                             |
| **Método**             | Función que pertenece a un objeto o clase.                                                         |
| **Objeto**             | Colección de pares clave-valor (propiedades y métodos). Ver módulo 06.                             |
| **Operador ternario**  | Forma compacta de un condicional: `condición ? siVerdadero : siFalso`.                             |
| **POO**                | Programación Orientada a Objetos: paradigma basado en clases y objetos. Módulo 07.                |
| **Polimorfismo**       | Capacidad de que distintos objetos respondan al mismo método de formas propias. Ver módulo 07.    |
| **Primitivo**          | Tipo de dato básico inmutable: `string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint`. |
| **Promise**            | Objeto que representa el resultado futuro de una operación asíncrona.                              |
| **Scope (ámbito)**     | Región del código donde una variable es accesible (global, de función, de bloque).                |
| **Template literal**   | Cadena con comillas invertidas que permite interpolación `${}` y multilínea. Ver módulo 06.       |
| **`var`**              | Forma antigua de declarar variables (ámbito de función); preferimos `let`/`const`.                |
